9-Edge Vision Model — Fine-tune 指南
====================================

你而家情況
----------
- 已升級 llava:13b，但 WOLF 仍讀唔到價（0/9）
- 原因：vision LLM 睇 TradingView 細字唔穩，唔係單純「再升級」就解決
- ETN 只有 1 個標準例子 → WOLF 冇專屬例子

三條路（由易到難）
------------------

【路線 1】Few-shot 例子庫（免費，你而家最應做）
  1. 每隻股評完後，用你認為正確嘅分數儲存：
     python add_training_example.py -i reports\WOLF_2026-06-20.png --interactive -s WOLF
  2. 目標：10-20 隻唔同股（ETN、WOLF、NVDA...）
  3. 再 run：python prepare_finetune_dataset.py
  唔改 model 權重，但同 symbol 會準好多

【路線 2】混合模式（推薦，我會加落 check_chart.py）
  - 價格 / 市值 / 均量 / RS → yfinance 自動查（準）
  - Ollama 只負責睇圖：形態、S&R、趨勢
  - 用真實價格驗證 AI，錯就 reject
  命令：python check_chart.py --paste --symbol WOLF --hybrid

【路線 3】真 Fine-tune LoRA（進階，8GB 顯存可做細 model）
  需要：
  - 30-50+ 張圖，每張人手標好 JSON（用 add_training_example.py）
  - Windows + CUDA + Python 3.10
  - LLaMA-Factory 或 unsloth（vision LoRA）

  步驟概要：
  1. 收集例子
     python add_training_example.py -i chart.png --interactive -s SYMBOL
     （重複 30-50 次，唔同股、唔同形態）

  2. 匯出數據集
     python prepare_finetune_dataset.py
     → finetune_data/9edge_train.jsonl

  3. 安裝 LLaMA-Factory（另一個 Python 環境建議）
     git clone https://github.com/hiyouga/LLaMA-Factory.git
     cd LLaMA-Factory && pip install -e .

  4. 用 LoRA fine-tune llava（7b 比 13b 易塞入 8GB）
     參考 LLaMA-Factory docs: LLaVA + mllm demo
     訓練 1-3 小時（視乎數據量）

  5. 匯出 GGUF → 匯入 Ollama
     ollama create 9edge-ft -f Modelfile
     （FROM 指向你匯出嘅 custom model）

  注意：Ollama 唔支援一鍵 fine-tune，要經 LLaMA-Factory 訓練後再匯入。

你部機（3070 Ti 8GB）建議
-------------------------
- llava:7b + LoRA：可行
- llava:13b + LoRA：好緊，可能 OOM
- 訓練時關閉其他食顯存程式

而家即刻可做（5 分鐘）
---------------------
1. 雙擊 rebuild_model.bat
2. 為 WOLF 加標準例子：
   python add_training_example.py -i reports\WOLF_2026-06-20.png --interactive -s WOLF
   （跟住我 WOLF_正確版.md 逐項輸入 y/n）
3. 再 run 貼圖分析.bat

長遠：路線 2（hybrid）+ 路線 1（例子庫）比純 fine-tune 更快見效。
